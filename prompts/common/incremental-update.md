# RSS 增量更新策略

## 更新流程

1. **读取现有 RSS**（如果存在 `output/SOURCE_NAME.xml`）
   - 提取所有已有条目的 `<guid>` 或 `<link>`
   - 建立已处理 ID/URL 集合

2. **去重新内容**
   - 只处理不在已有集合中的条目
   - 避免重复抓取和生成摘要

3. **合并内容**
   - 将新条目添加到已有条目前面
   - 保留最近 100 条（按 `<pubDate>` 排序）
   - 移除超过 100 条的旧条目

4. **输出 RSS**
   - 输出到 `output/SOURCE_NAME.xml`
   - 按发布时间倒序排列
   - 更新 `<lastBuildDate>` 为当前时间（ISO 8601 格式）
